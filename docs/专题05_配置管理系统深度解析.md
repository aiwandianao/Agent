## 一、真实业务场景

### 场景 1：多租户 SaaS 平台

**业务需求**：
- 不同租户需要不同的 LLM 配置
- 不同租户需要启用不同的 MCP 工具
- 不同租户需要不同的工作流配置

**对应技术**：
- 每个租户独立的 Client 配置
- 按租户隔离的 MCP 工具
- 配置化管理，无需修改代码

### 场景 2：AB 测试

**业务需求**：
- 测试不同的 Prompt 效果
- 测试不同的 LLM 模型
- 测试不同的工作流策略

**对应技术**：
- 多版本 Prompt 配置
- 动态切换模型
- 配置灰度发布

### 场景 3：动态扩展

**业务需求**：
- 新增 AI 模型提供商
- 新增 MCP 工具服务
- 新增工作流节点

**对应技术**：
- 配置驱动的自动装配
- 插件化的架构设计
- 热加载配置变更

## 四、Armory 自动装配机制

### 4.1 设计目标

**问题**：系统启动时如何自动创建并注册 ChatClient Bean？

**方案**：Armory 自动装配机制
- 配置驱动：通过配置文件指定需要装配的组件
- 责任链模式：按顺序装配 API → Model → MCP → Advisor → Client → Flow → Agent
- 缓存防重：使用 Redis Set 记录已装配的 ID

### 4.2 装配流程

```
系统启动
    ↓
读取 armory 配置
  - enable: true
  - clientIdList: [client_analyzer, client_performer]
  - agentIdList: [agent_article_writer]
    ↓
启动装配任务（异步）
    ↓
┌─────────────────────────────────────────────────────────────────────┐
│  责任链装配                                                          │
├─────────────────────────────────────────────────────────────────────┤
│  1. API 装配                                                         │
│     ├─ 从数据库加载 API 配置                                         │
│     ├─ 创建 OpenAiApi Bean                                          │
│     └─ 注册到 Spring 容器                                            │
│      ↓                                                               │
│  2. Model 装配                                                       │
│     ├─ 从数据库加载 Model 配置                                       │
│     ├─ 关联 API                                                     │
│     ├─ 创建 ChatModel Bean                                          │
│     └─ 注册到 Spring 容器                                            │
│      ↓                                                               │
│  3. MCP 装配                                                         │
│     ├─ 从数据库加载 MCP 配置                                         │
│     ├─ 创建 McpSyncClient                                           │
│     ├─ 初始化（获取工具列表）                                        │
│     └─ 注册到 Spring 容器                                            │
│      ↓                                                               │
│  4. Advisor 装配                                                     │
│     ├─ 加载 Advisor 配置（ChatMemory, RequestRetry）                │
│     └─ 注册到 Spring 容器                                            │
│      ↓                                                               │
│  5. Client 装配                                                      │
│     ├─ 从数据库加载 Client 配置                                       │
│     ├─ 关联 Model, API, Advisor                                     │
│     ├─ 创建 ChatClient Bean                                         │
│     └─ 注册到 Spring 容器（使用 client_id 作为 Bean 名）            │
│      ↓                                                               │
│  6. Flow 装配                                                        │
│     ├─ 从数据库加载 Flow 配置                                        │
│     └─ 加载 Prompt 内容                                             │
│      ↓                                                               │
│  7. Agent 装配                                                       │
│     ├─ 从数据库加载 Agent 配置                                        │
│     ├─ 关联 Flow 和 Client                                          │
│     └─ 注册到策略工厂                                               │
└─────────────────────────────────────────────────────────────────────┘
    ↓
装配完成，记录到 Redis（防重复）
    ↓
系统就绪，可以处理请求
```

### 4.3 核心代码

```java
@Component
public class ArmoryConfig {

    @Value("${armory.enable:false}")
    private boolean enable;

    @Value("${armory.client-id-list:}")
    private List<String> clientIdList;

    @Value("${armory.agent-id-list:}")
    private List<String> agentIdList;

    @PostConstruct
    public void assembleArmory() {
        if (!enable) {
            return;
        }

        log.info("开始自动装配 Armory...");

        // 使用责任链模式装配
        armoryChain.assemble(context -> {
            // 装配 Client
            for (String clientId : clientIdList) {
                assembleClient(clientId);
            }

            // 装配 Agent
            for (String agentId : agentIdList) {
                assembleAgent(agentId);
            }
        });

        log.info("Armory 自动装配完成");
    }

    private void assembleClient(String clientId) {
        // 1. 检查是否已装配
        if (redisService.hasClient(clientId)) {
            log.info("Client {} 已装配，跳过", clientId);
            return;
        }

        // 2. 加载配置
        AiClientVO clientVO = clientDao.queryByClientId(clientId);

        // 3. 装配 API
        OpenAiApi api = assembleApi(clientVO.getApiId());

        // 4. 装配 Model
        ChatModel model = assembleModel(clientVO.getModelId(), api);

        // 5. 装配 Advisor
        ChatMemoryAdvisor advisor = assembleAdvisor(clientVO.getAdvisorId());

        // 6. 创建 ChatClient
        ChatClient chatClient = ChatClient.builder(api)
            .defaultModel(model)
            .defaultAdvisors(advisor)
            .build();

        // 7. 注册到 Spring 容器
        registerBean(clientId, chatClient);

        // 8. 记录已装配
        redisService.addClient(clientId);

        log.info("Client {} 装配完成", clientId);
    }
}
```

## 五、AOP 缓存机制

### 5.1 设计目标

**问题**：配置数据频繁查询数据库，性能差

**方案**：AOP + Redis 缓存
- `@Cacheable`：查询时先查缓存
- `@CacheEvict`：更新时清除缓存

### 5.2 缓存注解

```java
// 缓存注解定义
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cacheable {
    String key();           // 缓存键，支持 SpEL 表达式
    int expire() default 3600;  // 过期时间（秒）
}

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface CacheEvict {
    String key();           // 要清除的缓存键
}
```

### 5.3 AOP 切面实现

```java
@Aspect
@Component
public class CacheableAspect {

    @Resource
    private RedisTemplate<String, Object> redisTemplate;

    @Around("@annotation(cacheable)")
    public Object around(ProceedingJoinPoint joinPoint, Cacheable cacheable) throws Throwable {
        // 1. 解析缓存键
        String key = parseKey(cacheable.key(), joinPoint);

        // 2. 查询缓存
        Object cached = redisTemplate.opsForValue().get(key);
        if (cached != null) {
            log.debug("缓存命中: {}", key);
            return cached;
        }

        // 3. 执行方法
        Object result = joinPoint.proceed();

        // 4. 写入缓存
        redisTemplate.opsForValue().set(key, result, cacheable.expire(), TimeUnit.SECONDS);
        log.debug("写入缓存: {}, 过期时间: {}秒", key, cacheable.expire());

        return result;
    }

    private String parseKey(String keySpEL, ProceedingJoinPoint joinPoint) {
        // 支持 #clientId, #agentId 等 SpEL 表达式
        // ...
        return parsedKey;
    }
}

@Aspect
@Component
public class CacheEvictAspect {

    @Resource
    private RedisTemplate<String, Object> redisTemplate;

    @After("@annotation(cacheEvict)")
    public void after(JoinPoint joinPoint, CacheEvict cacheEvict) {
        // 1. 解析缓存键
        String key = parseKey(cacheEvict.key(), joinPoint);

        // 2. 删除缓存
        redisTemplate.delete(key);
        log.debug("清除缓存: {}", key);
    }
}
```

### 5.4 使用示例

```java
@Service
public class ClientService {

    // 查询时自动缓存
    @Cacheable(key = "#clientId", expire = 3600)
    public AiClientVO queryByClientId(String clientId) {

    }

    // 更新时自动清除缓存

        // 更新数据库

        // 缓存自动清除
    }
```

## 七、配置热加载

### 7.1 设计目标

**问题**：修改配置后需要重启服务？

**方案**：配置热加载
- 监听配置变更
- 动态刷新 Spring Bean
- 无需重启服务

### 7.2 实现方式

```java
@Component
public class ConfigChangeListener {

    @EventListener
    public void onConfigChange(ConfigChangeEvent event) {
        String configType = event.getConfigType();
        String configId = event.getConfigId();

        switch (configType) {
            case "client":
                refreshClient(configId);
                break;
            case "mcp":
                refreshMcp(configId);
                break;
            case "prompt":
                refreshPrompt(configId);
                break;
        }
    }

    private void refreshClient(String clientId) {
        // 1. 清除缓存
        redisService.delete("client:" + clientId);

        // 2. 重新加载配置
        AiClientVO clientVO = clientDao.queryByClientId(clientId);

        // 3. 重新创建 ChatClient
        ChatClient newClient = createChatClient(clientVO);

        // 4. 更新 Spring Bean
        updateBean(clientId, newClient);

        log.info("Client {} 已刷新", clientId);
    }
}
```

**Q3**: AOP 缓存机制是如何工作的？

<details>
<summary>点击查看答案</summary>

**答案**：
**原理**：通过 AOP 拦截方法调用，在执行前后加入缓存逻辑

**流程**：
```
@Cacheable 方法调用
    ↓
1. 解析缓存键（如 #clientId）
    ↓
2. 查询 Redis
    ├─ 命中 → 直接返回缓存
    └─ 未命中 ↓
3. 执行方法（查询数据库）
    ↓
4. 写入 Redis（设置过期时间）
    ↓
5. 返回结果

@CacheEvict 方法调用
    ↓
1. 执行方法（更新数据库）
    ↓
2. 删除 Redis 缓存
    ↓
3. 返回
```

**优势**：
- 透明：业务代码无需关心缓存
- 自动：缓存和失效自动处理
- 灵活：通过注解配置缓存策略

</details>

**Q4**: Client、Model、API 之间是什么关系？

<details>
<summary>点击查看答案</summary>

**答案**：
```
API (提供商配置)
  ├─ api_id: api_openai
  ├─ api_base_url: https://api.openai.com/v1
  └─ api_key: sk-xxx
      ↓ 关联 (1:N)
Model (模型配置)
  ├─ model_id: model_gpt4
  ├─ api_id: api_openai
  └─ model_value: gpt-4
      ↓ 关联 (N:1)
Client (客户端配置)
  ├─ client_id: client_analyzer
  ├─ model_id: model_gpt4
  ├─ api_id: api_openai (通过 Model 关联)
  ├─ advisor_id: advisor_chat_memory
  ├─ temperature: 0.7
  └─ max_tokens: 2000
```

**关系说明**：
- 一个 API 可以有多个 Model（如 OpenAI 有 GPT-3.5、GPT-4）
- 一个 Model 只能属于一个 API
- 一个 Client 使用一个 Model
- 一个 Model 可以被多个 Client 使用

</details>

**Q10**: 如何实现配置的版本管理和回滚？

<details>
<summary>点击查看答案</summary>

**答案**：
**方案 1：配置快照表**
```sql
CREATE TABLE ai_config_snapshot (
    snapshot_id VARCHAR(50) PRIMARY KEY,
    config_id VARCHAR(50),
    config_type VARCHAR(20),    -- client, mcp, prompt
    config_content JSON,
    version INT,
    created_at TIMESTAMP,
    created_by VARCHAR(50)
);
```

**方案 2：保存快照**
```java
public void updateConfig(String configId, Config newConfig) {
    // 1. 保存旧版本快照
    Config oldConfig = configDao.getById(configId);
    saveSnapshot(configId, oldConfig);

    // 2. 更新配置
    configDao.update(configId, newConfig);
}

private void saveSnapshot(String configId, Config config) {
    String snapshotId = UUID.randomUUID().toString();
    int version = getNextVersion(configId);

    ConfigSnapshot snapshot = ConfigSnapshot.builder()
        .snapshotId(snapshotId)
        .configId(configId)
        .configContent(JSON.toJSONString(config))
        .version(version)
        .createdAt(LocalDateTime.now())
        .createdBy(getCurrentUser())
        .build();

    snapshotDao.insert(snapshot);
}
```

**方案 3：回滚**
```java
public void rollbackConfig(String configId, int targetVersion) {
    // 1. 获取目标版本快照
    ConfigSnapshot snapshot = snapshotDao.getByVersion(configId, targetVersion);
    Config config = JSON.parseObject(snapshot.getConfigContent(), Config.class);

    // 2. 恢复配置
    configDao.update(configId, config);

    // 3. 记录回滚操作
    logService.logRollback(configId, targetVersion, getCurrentUser());
}
```

**后台界面**：
- 版本列表
- 版本对比
- 一键回滚

</details>
